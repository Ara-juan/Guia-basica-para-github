Guía De GitHub Para Principiantes

¿Qué es GitHub?

Es una plataforma web donde puedes:

  • Alojar tus proyectos (Repositorios)
  • Trabajar en equipo con control de cambios
  • Reportar problemas (issues)
  • Revisar y fusionar código (pull requests)
  • Compartir tus proyectos con el mundo (Open source)

Pasos Iniciales En GitHub

  1. Crear una cuenta
    Haz clic aquí para registrarte
  2. Crea Tu Primer Repositorio

    Un repositorio GitHub es un espacio de almacenamiento para un proyecto, que puede contener archivos de código, imágenes o cualquier otro archivo necesario para que el proyecto funcione.

    Haz clic en el botón "+" y "New repository".

    Los repositorios pueden ser públicos (accesibles para cualquiera) o privados (limitados únicamente a contribuyentes invitados).

  3. Hacer un commit

    Un commit en GitHub es una instantánea del estado del proyecto, que guarda los cambios realizados en el repositorio local.

    1. Entrar al repositorio.
    2. Haz clic en cualquier archivo (Ejm: Readme.md).
    3. Pulsa el icono de lápiz para editarlo.
    4. Haz cualquier cambio.
    5. Al final de la página, pon un mensaje de commit (ej: actualizo la descripción).
    6. Pulsa Commit changes.

    Dentro del repositorio podemos ver los commits que se han hecho:

  4. Crear una rama

    Las ramas permiten hacer cambios sin afectar la versión principal del proyecto (main).

    1. En la parte superior del repo, busca un menú desplegable que diga main.
    2. Escribe un nuevo nombre (ej: rama-integración) y presiona Enter.
    3. Estás en una rama nueva. Haz cambios libremente.
  5. Abre y mergea una Pull Request

    Una pull request en GitHub es una funcionalidad que permite a los desarrolladores solicitar aprobación de cambios antes de integrarlos en la rama principal de un proyecto (main).

    Cuando termines de trabajar en tu rama:

    1. GitHub mostrará un botón para "Compare & Pull request".
    2. Escribe qué has cambiado.
    3. Puedes asignar revisores (opcional).
    4. Haz clic en Create pull request.
    5. En la sección de Pull Requests, haz clic en Merge pull request.
